# AI Coding Agent 三框架对比评估 — 最终报告

> **评估日期**: 2026-04-15
> **统一模型**: claude-sonnet-4-5-20250929 (via api.dbh.baidu-int.com)
> **评估任务**: 4 大类 11 个子任务（1 个跳过）
> **项目仓库**: [luxuriance19/agent-eval-arena](https://github.com/luxuriance19/agent-eval-arena)

---

## Executive Summary

本次评估在**完全相同的底层模型**（Claude Sonnet 4.5）下，对比了三个 AI 编程 Agent 框架的综合能力。通过代码编写、浏览器操作、网页监控和真实交互四大类任务，我们发现：

**Hermes Agent 以 7.84 分位居第一**，在浏览器操作（8.89）和网页监控（8.92）两个类别中均取得最高分，且是唯一一个**零重试**的 Agent——所有 10 个有效任务均一次完成。

**Claude Code 以 7.71 分紧随其后**，在真实交互类别中表现最好（6.13），其 GitHub Discussion 内容质量和 PR 尝试深度均领先。但受 429 rate limiting 和 Cloudflare 拦截影响，效率分被明显拉低。

**OpenClaw 以 5.95 分位列第三**，其 Cat4 全部失败（0 分）是最大拖累因素。context overflow 和 SSRF 策略限制导致多个任务需要重试。但值得注意的是，OC 是唯一成功完成 worldtimeapi.org 端点的 Agent，且 Cat1 代码项目中效率最高（179s）。

## 各类别深度分析

### Category 1: 代码项目（CLI 书签管理器 "bmk"）

三个 Agent 都成功完成了完整的 Python CLI 项目，包括 SQLite 存储、9 个子命令、标签系统和测试套件。

| 指标 | CC | OC | HA |
|------|-----|-----|-----|
| 测试通过 | 20/20 | 20/20 | 14/14 |
| 耗时 | 268s | **179s** | 343s |
| 类型注解 | 现代 PEP 604 | 旧式 typing.List | 现代 + Py3.9 兼容 |
| README | 无 | 有 | 极简 |
| 加权分 | 7.75 | **8.00** | 7.80 |

**关键发现**: OC 生成速度最快但代码质量（旧式 typing、HTML XSS 风险）不如其他两家。HA 主动适配了 Python 3.9 兼容性，体现出更好的工程意识。

## 关键发现

### 1. 底层模型相同，框架差异显著

三个 Agent 使用完全相同的 Claude Sonnet 4.5 模型，但最终得分差距达 1.89 分（7.84 vs 5.95）。这证明了 **Agent 框架的编排能力、工具集成、错误恢复策略**对最终表现的影响远大于预期。

### 2. 稳定性是最大区分因素

- **Hermes**: 10/10 有效任务一次完成（100% 首次成功率）
- **CC**: 8/10 一次完成，1 次 429 重试（80%）
- **OpenClaw**: 约 5/10 一次完成，多次 context overflow（~50%）

OC 的 context overflow 问题在长任务中反复出现，严重影响了效率分和自主性分。

### 3. 真实交互是当前 Agent 的天花板

Cat4 是所有 Agent 表现最差的类别。向开源仓库提交有意义的 PR（4C）要求 Agent 具备：理解陌生代码库 → 识别真实问题 → 创建修复 → 管理 git 操作 → 与远程仓库交互。这一多步骤工作流目前超出了所有测试 Agent 的能力范围。

### 4. 浏览器操作已相当成熟

Cat2 是三个 Agent 平均得分最高的类别。通过 agent-browser/Playwright 等工具，表单填写、数据提取、多步导航都能高质量完成。这表明浏览器自动化已成为 AI Agent 的基本能力。

### 5. 错误恢复策略各有千秋

- **CC**: 创造性解决问题（Web Archive 绕过 Cloudflare）但耗时长
- **OC**: 自动 context compaction 但成功率不稳定
- **HA**: 完整记录错误+优雅降级（worldtime 失败但记录了重试详情和错误原因）

## 建议

### 选择建议

| 场景 | 推荐 Agent | 原因 |
|------|-----------|------|
| 需要高可靠性的自动化任务 | **Hermes Agent** | 零重试，自主性最强 |
| 需要高质量报告/文档生成 | **Claude Code** | Executive Summary 质量最高 |
| 需要快速代码生成 | **OpenClaw** | Cat1 最快（179s） |
| 浏览器操作密集型任务 | **Hermes Agent** | Cat2 最高分 8.89 |
| 需要与外部服务交互 | **Claude Code** | Cat4 唯一达到 6+ 分 |

### 改进方向

- **OpenClaw**: 需要解决 context overflow 问题，改善 `gh` CLI 集成
- **Claude Code**: 需要更好的 rate limit 处理策略，避免长时间绕行
- **Hermes Agent**: 可以加强代码生成速度（Cat1 最慢），以及真实交互能力
- **所有 Agent**: 多步骤真实世界交互（PR 贡献）是共同的改进方向
