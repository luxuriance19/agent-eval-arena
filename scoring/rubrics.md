# 评分标准 (Rubrics)

## 通用评分维度

### 正确性 (Correctness) — 权重 30%

| 分数 | 描述 |
|------|------|
| 0-2 | 核心功能不工作，输出错误 |
| 3-4 | 部分正确，但关键功能有明显 bug |
| 5-6 | 大部分正确，有小错误不影响使用 |
| 7-8 | 全部正确，极少数边界情况未覆盖 |
| 9-10 | 完美正确，包含边界处理 |

### 完整性 (Completeness) — 权重 25%

| 分数 | 描述 |
|------|------|
| 0-2 | 缺失 >50% 要求的功能 |
| 3-4 | 完成 50-70% 的要求 |
| 5-6 | 完成 70-85% 的要求 |
| 7-8 | 完成 85-95% 的要求 |
| 9-10 | 100% 完成，可能有额外功能 |

### 质量 (Quality) — 权重 20%

| 分数 | 描述 |
|------|------|
| 0-2 | 无类型注解、无文档、代码混乱 |
| 3-4 | 基本可读，缺少类型和文档 |
| 5-6 | 结构合理，部分有类型和文档 |
| 7-8 | 良好的代码组织、类型完整、有文档 |
| 9-10 | 地道风格、优秀的抽象和命名 |

### 效率 (Efficiency) — 权重 15%

| 分数 | 描述 |
|------|------|
| 0-2 | 超时或 token 消耗异常高，大量无效重试 |
| 3-4 | 完成但效率低，很多不必要操作 |
| 5-6 | 中等效率，有些冗余步骤 |
| 7-8 | 高效完成，少量冗余 |
| 9-10 | 极高效，直达目标 |

### 自主性 (Autonomy) — 权重 10%

| 分数 | 描述 |
|------|------|
| 0-2 | 多次需要人工干预才能继续 |
| 3-4 | 需要 1-2 次人工干预 |
| 5-6 | 自主完成但遇到错误需要多次重试 |
| 7-8 | 自主完成，遇到错误能自我恢复 |
| 9-10 | 完全自主，零干预零错误 |

---

## Category 专项标准

### Cat1: 代码项目

- **正确性**: 所有 9 个 CLI 命令是否按预期工作
- **完整性**: pyproject.toml、src 布局、所有命令、测试文件是否齐全
- **质量**: Python 3.12 风格、类型注解、docstrings、错误处理
- **效率**: 是否一次性创建所有文件并通过测试
- **自主性**: 测试失败后是否自行修复

### Cat2: 浏览器操作

- **正确性**: 目标数据是否准确提取，表单是否正确填写
- **完整性**: 所有截图是否都有，JSON 输出是否完整
- **质量**: 数据结构是否清晰，截图是否有效
- **效率**: 浏览器操作是否流畅，是否有不必要的页面加载
- **自主性**: 遇到弹窗/CAPTCHA/加载延迟是否自行处理

### Cat3: 网页监控

- **正确性**: 数据是否准确，报告分析是否正确
- **完整性**: 所有时间维度/数据字段是否覆盖
- **质量**: Markdown 报告格式、分析深度
- **效率**: 是否合理并行抓取
- **自主性**: 网站加载失败是否有 fallback

### Cat4: 真实交互

- **正确性**: 帖子/PR 是否成功创建
- **完整性**: 内容长度、格式、元数据是否达标
- **质量**: 文章质量、PR 贡献的真实价值
- **效率**: 是否快速找到目标并执行
- **自主性**: 遇到权限/认证问题是否自行解决
